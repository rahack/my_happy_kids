# Happy Kids — мотивационный бот для детей на каникулах

Прототип Telegram Mini App: админ (родитель) ведёт список ежедневных задач и наград для детей, отмечает выполнение и вручает награды по заполнению шкалы прогресса.

## Стек
- Node.js + Express
- SQLite (better-sqlite3)
- Telegraf (Telegram bot)
- Vanilla JS Mini App (статика из `public/`)

## Структура
```
.
├── server.js        # Express + сессии + REST API + автозапуск туннеля
├── bot.js           # Telegram-бот, кнопка запуска Mini App
├── tunnel.js        # Wrapper над cloudflared (quick tunnel)
├── db.js            # SQLite, схема и создание дефолтного админа
├── public/          # Mini App: index.html, app.js, style.css
├── tools/
│   └── cloudflared.exe  # бинарь cloudflared (Windows AMD64)
└── .env             # TELEGRAM_TOKEN, WEBAPP_URL?, SESSION_SECRET?
```

## Запуск
1. Зависимости установлены (`npm install` если нужно повторить).
2. `.env`:
   ```
   TELEGRAM_TOKEN=<токен от BotFather>
   SESSION_SECRET=<любая строка>     # опционально
   PORT=3000                         # опционально
   # WEBAPP_URL=https://...          # опционально, если хотите свой URL
   # NO_TUNNEL=1                     # опционально, отключает автозапуск туннеля
   ```
3. Старт:
   ```
   npm start
   ```
   `server.js` автоматически:
   - запустит `tools/cloudflared.exe` (quick tunnel, без регистрации);
   - получит публичный URL вида `https://<random>.trycloudflare.com`;
   - подставит его как `WEBAPP_URL` для бота.

   В консоли увидите строки `[tunnel] public URL ready: ...` и `[bot] @<...> ready`.
4. Открыть Mini App:
   - В Telegram: написать боту `/start` → нажать «Открыть админку».
   - Или в браузере: <http://localhost:3000> (туннель не нужен для локального доступа).
5. Войти: `admin` / `admin` (поменять можно в «Настройках»).

### Туннелирование
- По умолчанию используется **cloudflared quick tunnel** (бесплатно, без регистрации). Бинарь лежит в `tools/cloudflared.exe`.
- URL **меняется при каждом перезапуске** — это особенность quick tunnel.
- Чтобы зафиксировать URL: создайте named tunnel в кабинете Cloudflare и пропишите его в `WEBAPP_URL` (тогда автозапуск пропустится).
- Кастомный путь к бинарю: `CLOUDFLARED_PATH=C:\path\to\cloudflared.exe`.
- Отключить туннелирование: `NO_TUNNEL=1` (полезно, если хочется только локальный доступ).

## Сценарий

### Настройка (режим «Родитель 🔓»)
1. Залогиниться как `admin` / `admin` (поменять пароль можно в «Настройках»).
2. Переключиться в режим **«Родитель 🔓»** (кнопка справа вверху) — потребуется повторный ввод пароля.
3. Создать профиль ребёнка (имя, возраст, пол, опционально фото).
4. Открыть профиль → выбрать день (полоса под шкалой) → добавить задачи → назначить награду.
5. Переключиться в режим **«Просмотр 👀»** и передать устройство ребёнку.

### Выполнение (режим «Просмотр 👀»)
1. Ребёнок отмечает выполненные задачи галочкой → шкала заполняется.
2. Пока не 100% — на месте награды 🎁 «Сюрприз!». Название не светится.
3. На 100% появляется кнопка **«Открыть награду»**.
4. Родитель нажимает → форма **логин+пароль** → «Сюрприз готов!» (название всё ещё скрыто) + кнопка «Вручить награду».
5. Клик по **«Вручить награду»** → 🏆 + название награды наконец-то раскрывается. Ребёнок видит свой приз.

### Просмотр других дней
Полоса дат под шкалой прогресса:
- **Прошлое** — read-only архив с маркерами выполнения и историей наград.
- **Сегодня** — обычный поток.
- **Будущее** — для ребёнка скрыто, родитель может планировать задания и награду заранее.

### Защитные механики
- Снятие галочки на сервере автоматически сбрасывает `claimed=0` → ребёнок не «зафиксирует» награду навсегда снятием+постановкой.
- Изменение названия награды в режиме родителя тоже сбрасывает `claimed=0` — это новая награда, надо заслужить.
- Раскрытие награды требует пароль каждый раз при заходе в профиль (состояние не персистится).
- Переключение из «Просмотр» в «Родитель» требует ввода пароля.

## Два режима интерфейса

Кнопка переключения справа вверху, выбор сохраняется в `localStorage`.

| | Просмотр (по умолчанию) | Родитель |
|---|---|---|
| Отметка задач (галочки на сегодня) | ✅ | ✅ |
| Шкала прогресса + календарь дней | ✅ | ✅ |
| Просмотр прошедших дней (архив) | ✅ | ✅ |
| Раскрытие награды (с паролем + кнопкой «Вручить») | ✅ | — (нет кнопки, чтобы сюрприз показал ребёнок) |
| Видеть название награды до вручения | ❌ | ✅ (планирование) |
| Планирование на будущие даты | ❌ | ✅ |
| Создание/редактирование/удаление детей и фото | ❌ | ✅ |
| Добавление/удаление задач | ❌ | ✅ (сегодня + будущее) |
| Назначение/изменение награды | ❌ | ✅ |
| Статистика и история | ❌ | ✅ |
| Настройки, выход | ❌ | ✅ |

Переключатель — UI-удобство, не серверная защита (один аккаунт). Но переход «Просмотр → Родитель» **требует пароля** (повторная проверка через `/api/verify-admin`), чтобы ребёнок не переключился сам.

## API (для разработки)
- `POST /api/login {username, password}` / `POST /api/logout` / `GET /api/me`
- `POST /api/verify-admin {username, password}` — повторная проверка без смены сессии (раскрытие награды, вход в режим Родителя)
- `POST /api/change-password {oldPassword, newPassword}`
- `GET /api/kids` / `POST /api/kids {name, age, gender, photo?}` / `PUT /api/kids/:id {name, age, gender, photo?}` / `DELETE /api/kids/:id`
- `GET /api/kids/:id` — профиль: `today`, `history`, `stats`, `calendar` (маркеры активности для полосы дат)
- `GET /api/kids/:id/day/:date` — задачи и награда за произвольную дату (используется календарём)
- `POST /api/kids/:id/tasks {date?, title}` / `POST /api/tasks/:id/toggle` / `DELETE /api/tasks/:id`
  - `toggle` автоматически сбрасывает `claimed=0` у награды дня, если после переключения задачи не все выполнены
- `POST /api/kids/:id/reward {date?, title}` (upsert; смена title сбрасывает `claimed=0`)
- `POST /api/rewards/:id/claim` (на сервере проверяет, что все задачи дня выполнены)

Фото в `kids.photo` — base64 data URL; клиент даунскейлит до 256×256 (JPEG, q≈0.85). Лимит JSON-body поднят до 2МБ.

## Известные ограничения и нюансы
- Telegram WebApp `initData` не валидируется на сервере — авторизация по логину/паролю и сессии (как было в ТЗ).
- Один админ (id=1), пароль хранится bcrypt-хэшем.
- БД-файл `data.db` создаётся автоматически при первом запуске.
- Quick tunnel URL **меняется при каждом перезапуске**. Бот автоматически переустанавливает Menu Button с новым URL.
- На фронте стоит `Cache-Control: no-store` + cache-buster в URL — Telegram очень любит кешировать Mini App'ы. Если после рестарта в Telegram видите старое — закройте/откройте чат с ботом или Reload Page внутри Mini App.

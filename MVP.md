# MVP

## Исходное ТЗ

Нужно создать телеграмм приложение. Токен для бота указан в `.env`.
Приложение для мотивации детей на летних каникулах.

1. Каждодневный список задач и каждодневная награда за выполнение.
2. Есть всего один пользователь с полными правами, который администрирует список задач по дням и награды. Его можно создать пока с логином `admin` и паролем `admin`. Также должна быть возможность менять пароль.
3. Администратор умеет создавать профили детей.
4. Назначать список задач на день и награду в конце выполнения всех задач на день каждому ребенку.
5. При создании профиля ребенка необходимо ввести имя, возраст, пол.
6. Каждое задание имеет название.
7. Каждая награда имеет название.
8. Администратор открывает профиль ребенка и видит задания прошлых дней и статистику выполнения и видит задания на текущий день.
9. Когда ребенок выполнил задание, администратор это отмечает.
10. Во время выполнения заданий появляется шкала.
11. После выполнения каждого задания шкала заполняется.
12. После заполнения на 100% появляется награда в профиле ребенка.

## Статус: реализовано

| # | Пункт | Где |
|---|---|---|
| 1 | Ежедневный список задач и награда | `tasks` и `rewards` с `date` по дням |
| 2 | Один админ `admin/admin`, смена пароля | `db.js` сидинг, `/api/change-password` |
| 3 | Создание профилей детей | `/api/kids` POST |
| 4 | Назначение задач и награды на день для ребёнка | `/api/kids/:id/tasks`, `/api/kids/:id/reward` |
| 5 | Имя, возраст, пол при создании | форма в `public/app.js` (renderKidsList) |
| 6, 7 | Название у задачи и награды | поля `title` в таблицах |
| 8 | Прошлые дни + статистика + текущий день | `getKidProfile` в `server.js` |
| 9 | Админ отмечает выполнение | `POST /api/tasks/:id/toggle` |
| 10, 11 | Шкала прогресса | `.progress-bar` в `style.css`, `pct = done / total` |
| 12 | Награда на 100% | `renderRewardSection` в `public/app.js` |

## Решения по архитектуре

- **Формат:** Telegram Mini App (WebApp), а не «голый» бот. Бот выступает лаунчером для админ-панели через Menu Button.
- **Хранилище:** SQLite (`better-sqlite3`). Один файл `data.db`, журналирование WAL.
- **Аудитория:** Только админ (родитель). У детей нет отдельного входа в бота — родитель сидит с ребёнком и отмечает галочки.
- **Туннелирование:** автозапуск `cloudflared` quick tunnel для HTTPS-URL (обязателен для Telegram Mini App).
- **Стек фронта:** vanilla JS без сборщиков — минимум зависимостей, мгновенная итерация.

## Расширения сверх ТЗ (по итогам обсуждения)

1. **Два режима интерфейса** (`localStorage.mode`):
   - **«Просмотр»** (по умолчанию) — родитель + ребёнок: только галочки и шкала. Скрыто всё добавление/удаление.
   - **«Админ»** — управление профилями, задачами, наградами, настройки.

2. **Скрытая награда с разблокировкой по паролю:**
   - Пока не все задачи выполнены — карточка «🔒 ???», название не светится.
   - На 100% — кнопка «Открыть награду» → встроенная форма логин+пароль → проверка через `POST /api/verify-admin` (новый эндпоинт, не меняет сессию) → название раскрывается и появляется «Вручить награду».
   - При снятии галочки задачи награда снова блокируется (`!allDone` проверяется до `claimed`).
   - Изменение названия награды автоматически сбрасывает `claimed=0` (переименование = новая награда).

3. **Анти-кеш Mini App:** `Cache-Control: no-store` + cache-buster в URL Menu Button.

4. **Тёмная тема Telegram:** зафиксированы цвета текста на светлых плашках, чтобы не было «белый на белом».

## Известные ограничения прототипа

- Telegram WebApp `initData` не валидируется на сервере — авторизация по логину/паролю и сессии, как было в ТЗ.
- Переключатель режима «Просмотр»/«Админ» — это UI-удобство, а не серверная защита (один аккаунт админа). Для защиты от случайного нажатия ребёнком — достаточно; для защиты от мотивированного — нет.
- Quick tunnel URL меняется при каждом перезапуске сервера (для стабильного URL нужен named tunnel в Cloudflare).
- Нет миграций БД, нет тестов, нет CI.

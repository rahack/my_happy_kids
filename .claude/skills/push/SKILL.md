# Skill: push

Пушит текущую ветку в `origin` репозитория `my_happy_kids`.
Используй этот скил когда пользователь просит «запушь», «отправь на GitHub», «push».

## Разрешения

```yaml
tools:
  exec: allow
```

## Шаги

### 1. Проверить состояние

```bash
git status
git log origin/master..HEAD --oneline
```

Если `git log origin/master..HEAD` пуст — нет новых коммитов для пуша. Сообщить пользователю и остановиться.

Если есть незакоммиченные изменения — **не пушить**, предложить сначала закоммитить (скил `commit`).

### 2. Запушить

```bash
git push origin master
```

(Ветка — `master`, репо — `git@github.com:rahack/my_happy_kids.git`)

### 3. Отчёт

Показать пользователю:
- Успешный пуш: ветку и хэш последнего коммита
- При ошибке — полный вывод git, возможные причины (нет SSH-ключа, remote ahead и т.д.)

## Правила

- Никогда не делать `--force` и не перезаписывать историю без явного подтверждения пользователя
- Не менять git config
- Ветка всегда `master` (проверить через `git branch --show-current` на всякий случай)
